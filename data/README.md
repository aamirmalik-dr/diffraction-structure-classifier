# Data

## What is committed

`data/sample/` holds six small synthetic diffraction patterns, one per
structure type (`sc`, `bcc`, `fcc`, `diamond`, `rocksalt`, `hcp`), each a
compressed `.npz` with the noisy pattern image, its radial profile, the exact
label, the rendered spot list, and the full generative metadata (zone axis,
lattice parameter, dose, orientation). They are produced by

```
crystalclass samples
```

Every sample is **synthetic**. It is generated by this repository's kinematical
diffraction simulator (`crystalclass.sim`), not measured on any instrument.
Nothing here is derived from experimental data, and there are no external
datasets, licensed or otherwise, anywhere in this project.

## No committed real image

Unlike a repository that ships a licensed experimental micrograph, this project
does **not** commit a real diffraction pattern. A correctly attributed,
clearly Creative-Commons-licensed experimental electron diffraction pattern was
not sourced for this build, so rather than commit something with an ambiguous
license, the real-data path is documented below and left for you to supply your
own image. The models never saw real data during training in any case; see the
domain-gap note in `models/MODEL_CARD.md`.

## Bring your own diffraction pattern

The pattern CNN runs on any single-crystal zone-axis diffraction image:

```
crystalclass classify your_pattern.png --method pattern
```

For a sensible result the image must satisfy the conditions in
`src/crystalclass/real.py`:

- the direct (undiffracted) beam sits at the image centre (pass a `center`
  argument to `load_diffraction_image` if it does not);
- the whole ring pattern is visible and roughly centred, since the model reads
  ring geometry;
- it is a diffraction pattern, not a real-space or bright-field image.

There is no ground truth for an arbitrary image, so the output is a predicted
structure and class probabilities, not an accuracy. Treat it as a starting
point that needs validation against a trusted reference, and expect the
synthetic-to-real domain gap described in the model card.
